# 🐾 Hipermascota Rafaela — E-Commerce de Accesorios para Mascotas

Bienvenido al repositorio oficial de **Hipermascota Rafaela**, una plataforma web moderna, rápida y totalmente accesible diseñada para la visualización, gestión y venta de accesorios para mascotas. El sistema permite a los usuarios explorar un catálogo dinámico, armar su carrito de compras y finalizar sus pedidos enviando un mensaje automatizado y detallado directamente a la línea de WhatsApp del local.

Además, cuenta con un panel de administración restringido (`/admin`) para la gestión del inventario y el control interno de los pedidos.

---

## 🚀 Stack Tecnológico

El proyecto está diseñado para ser ultraeficiente, seguro y escalable, utilizando las siguientes tecnologías:

* **Frontend:** TypeScript + Next.js (App Router)
* **Estilos y UI:** Tailwind CSS + Lucide Icons (iconografía limpia y accesible)
* **Backend & Base de Datos:** [Supabase](https://supabase.com/) (PostgreSQL para almacenamiento de datos, Supabase Auth para el panel de administración, y Supabase Storage para imágenes de productos).
* **Despliegue y CI/CD:** [Vercel](https://vercel.com/) (Integración continua directa con este repositorio de GitHub).
* **Asistente de Desarrollo:** Claude Code (utilizado para la generación, auditoría y optimización del código).

## 🛠️ Estructura y Módulos de la Aplicación

### 1. Área Pública (Cliente)
* **Inicio (Landing Page):** Presentación del negocio, banners promocionales de temporada, sección de categorías populares y carrusel de productos destacados.
* **Productos (Catálogo):**
    * Visualización en cuadrícula (Grid) adaptativa de todos los accesorios disponibles.
    * Filtros dinámicos interactivos por **Categoría**.
    * Ordenamiento por **Precio** (Menor a Mayor / Mayor a Menor).
    * Tarjetas de producto con detalles completos, precio, estado de stock y botón rápido de "Añadir al carrito".
* **Contacto:** Información institucional de la tienda, ubicación física en Rafaela, horarios de atención y accesos rápidos a redes sociales.
* **Carrito de Compras:**
    * Desplegable lateral (Sidebar) o página dedicada para revisar los productos seleccionados.
    * Modificación de cantidades y eliminación de ítems con recálculo de subtotales y total en tiempo real.
    * **Checkout sin Pasarela de Pago:** Botón principal "Confirmar Pedido por WhatsApp" que procesa el carrito, genera un texto perfectamente formateado (con saltos de línea, emojis y totales) y redirige al número oficial.

### 2. Panel de Administración (`/admin`)
* **Autenticación Segura:** Control de acceso mediante login gestionado por Supabase Auth para garantizar que solo el dueño del negocio pueda ingresar.
* **Gestión de Inventario (CRUD/ABM):**
    * **Alta:** Formulario para subir nuevos accesorios (Nombre, Descripción, Precio, Categoría, Stock y subida de imagen a Supabase Storage).
    * **Baja:** Eliminación lógica o física de productos que ya no se comercializan.
    * **Modificación:** Actualización rápida de precios, stock o descripciones desde una tabla interactiva.
* **Historial de Pedidos:** Registro interno automatizado en la base de datos de cada carrito que se envía por WhatsApp. Permite al administrador consultar qué se pidió, cuándo, y cambiar su estado de control interno (ej: *Pendiente*, *Entregado*).

## 📊 Modelo de Datos (Supabase SQL)

Para inicializar la base de datos en el editor SQL de Supabase, ejecuta el siguiente script que define la estructura exacta, las relaciones y las restricciones de integridad:

```sql
-- 1. Tabla de Categorías
CREATE TABLE categories (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    name TEXT NOT NULL UNIQUE,
    slug TEXT NOT NULL UNIQUE,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- 2. Tabla de Productos
CREATE TABLE products (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    name TEXT NOT NULL,
    description TEXT,
    price NUMERIC(10, 2) NOT NULL CHECK (price >= 0),
    category_id BIGINT REFERENCES categories(id) ON DELETE SET NULL,
    image_url TEXT,
    stock INT NOT NULL DEFAULT 0 CHECK (stock >= 0),
    is_featured BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- 3. Tabla de Pedidos (Historial para el Admin)
CREATE TABLE orders (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    customer_phone TEXT,
    items JSONB NOT NULL, -- Almacena un array de objetos [{product_id, name, qty, price}]
    total_amount NUMERIC(10, 2) NOT NULL,
    status TEXT DEFAULT 'Pendiente' CHECK (status IN ('Pendiente', 'Entregado', 'Cancelado')),
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);
```

## 📱 Especificación de Integración con WhatsApp

La API de WhatsApp se utiliza de manera directa sin intermediarios. La URL de destino debe respetar rigurosamente la siguiente configuración:

* **Número de Destino:** `5493492330921` (Formato internacional estricto para Rafaela, Argentina).
* **Construcción del Mensaje:** El texto se codifica mediante `encodeURIComponent()` para preservar los espacios y saltos de línea (`\n`).
* **Formato del Mensaje Sugerido:**
    ```text
    🐾 ¡Hola Hipermascota! Quiero realizar el siguiente pedido:
    
    -----------------------------------------
    • 2x Collar Reflectivo Rojo ($1.500 c/u)
    • 1x Juguete Mordillo Goma ($2.300 c/u)
    -----------------------------------------
    💰 Total Pedido: $5.300
    
    ¿Me confirman la disponibilidad para coordinar el retiro/envío? ¡Muchas gracias!
    ```

## ⚙️ Configuración del Entorno (`.env.local`)

Crea un archivo `.env.local` en la raíz de tu entorno local (este archivo está ignorado en `.gitignore` para proteger las credenciales). Inserta las variables de conexión provistas por los paneles de Supabase y Vercel:

```env
# Variables Públicas de Supabase (Acceso Frontend)
NEXT_PUBLIC_SUPABASE_URL=[https://tu-proyecto-id.supabase.co](https://tu-proyecto-id.supabase.co)
NEXT_PUBLIC_SUPABASE_ANON_KEY=tu-clave-anonima-jwt-aqui

# Número oficial de WhatsApp para redirección
NEXT_PUBLIC_WHATSAPP_NUMBER=5493492330921
```

## 🏃‍♂️ Instalación y Desarrollo Local

1.  **Clonar este repositorio:**
    ```bash
    git clone [https://github.com/geromendez1999/Hipermascota-Rafaela.git](https://github.com/geromendez1999/Hipermascota-Rafaela.git)
    cd Hipermascota-Rafaela
    ```

2.  **Instalar todas las dependencias requeridas:**
    ```bash
    npm install
    ```

3.  **Iniciar el entorno de desarrollo local:**
    ```bash
    npm run dev
    ```
    El servidor local se levantará de forma automática en [http://localhost:3000](http://localhost:3000).

## 🌐 Flujo de Despliegue Continuo (CI/CD)

El proyecto está óptimamente configurado para desplegarse de manera automática en **Vercel**:
1.  **Trigger de Producción:** Cada vez que se realiza un `git push` o se fusiona un Pull Request hacia la rama `main`, Vercel detecta el cambio de forma automática.
2.  **Etapa de Build:** Vercel compila la aplicación Next.js y TypeScript de forma estática y optimizada.
3.  **Inyección de Credenciales:** Las variables de entorno (`NEXT_PUBLIC_SUPABASE_URL` y `NEXT_PUBLIC_SUPABASE_ANON_KEY`) deben configurarse una única vez en el panel de control de Vercel (Settings > Environment Variables) utilizando las credenciales reales de producción de la cuenta de **Hipermascota**.
